The project

Creating a Recipe App

The challenge was to create a new recipe web app. It was necessary to find the current market weaknesses and what user needs are not met yet. From there, I had to ideate the new app.

My role:

UX and UI Designer

Objective:

Research the current recipe web app market and create a new web app that satisfies the user needs.

The Design process

Market and user research

Users are not searching for healthy meals, but quick and tasty ones

I started this process through research, analysing competitors and conducting user interviews. The competitors had many different features, even meal planners. My hypothesis was users wanted a recipe app that would allow them to cook healthy and balanced dishes.

However, from my interviews it seemed the users were searching for a simple app that would allow them to find quickly something they feel like eating at the moment, without spending much time cooking. With my findings, I created 3 user personas.

Defining the app and its functions

Providing a Taste filter when the user is looking for a recipe

I've created the MVP and the Jobs To Be Done. The main theme of the app will be "Quick & Tasty", with filters allowing the user to search through the recipes according to the taste. This feature together with a curated recipe collection with few easy-to-find ingredients would probably suit my users the most.

One of the Jobs To Be Done was particularly challenging. It was about providing to the users a simple way of finding balanced recipes.

Usually each recipe shows nutritional values and with some knowledge users can make a balanced meal by matching different dishes. This direction would require both knowledge from the user and time, so it wasn't what my users needed. I decided a good idea could have been to provide some menus options of balanced meals, so the user would only have to choose one and follow the recipe steps. The menus would change every day and be curated by experts.

Making the User flows

How would users achieve their goals?

The next step was creating the user flows.

User story 1

" When searching for a recipe, I want to be able to filter the results by taste, country, difficulty and time, so I can see only the recipes I am interested in. "

Entry point: Landing page

Success criteria: View a list of recipes according to the selected filters.

Task analysis:

  1. Landing page
  2. Click on a food category
  3. Food category page
  4. Click on the filter icon
  5. Select the wanted filters
  6. Click Ok
  7. Get a list of filtered recipes
Wireframing and User tests

Pain points of the usability test

After making my app wireframes, I created a prototype and ran a few usability tests. I was glad I've found different pain points in my app and was able to solve them early on in the designing process.

1

A big pain point of the app was the location of the My recipe section and how it was structured. The My recipe section allows users to add their own recipes to the App and also to check their liked recipes, all in one place.

The categories under the My recipes section have been deleted, making the interface cleaner.

2

Many users were confused while trying the app and did not understand immediately what kind of app they were using.

Now the home page has a brief description of the app.

A/B Preference test

Preference Testing

As I worked on my screens, I found out there is not a standard way of displaying nutritional information around the web. In UK there is the traffic lights system, however it isn't widely recognised. I therefore designed 2 different ways of showing such data and run a user preference test using the platform Usability Hub.

60% of the participants liked more Choice 1. From this test I also understood participants preferred to have less categories, so I decreased the categories to 5.
